# learning web security

### Login
- Session
- CSRF Token
    - a secret, unique value generated by the server for each user session.
    - purpose: prevent CSRF attacks
- Cookie
    - a small piece of data stored in the user's browser by ther website.
    - porpose: keep track of sessions, preferences, login status etc.
    - setting:
        - HTTP only: prevent XSS 
        - secure: prevent sniffing
        - samesite: prevent CSRF
- HTTPS
    - encrypts the communication between the user's browser and the server
    - purpose: prevent man-in-the-middle attacks
